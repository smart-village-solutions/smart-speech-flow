# Security Policy

## Supported Versions

This project currently provides security updates for the latest maintained release line and the current development branch.

| Version | Supported |
| ------- | --------- |
| `main` | :white_check_mark: |
| `1.1.x` | :white_check_mark: |
| `1.0.x` | :x: |
| `< 1.0` | :x: |

Older versions may contain known issues that will not be patched. If you are running an unsupported version, please upgrade before requesting a security fix.

## Reporting a Vulnerability

Please do **not** report security vulnerabilities through public GitHub issues, discussions, or pull requests.

Use GitHub's private vulnerability reporting for this repository instead:

- Go to the repository's **Security** tab
- Open a new **Private vulnerability report**
- Include affected version(s), reproduction steps, impact, and any suggested mitigation

If private reporting is unavailable for any reason, open a minimal issue asking the maintainers for a private contact channel and do not include exploit details.

## What to Expect

- We aim to acknowledge new reports within **72 hours**
- We aim to provide a status update within **7 days**
- We will validate the report, assess impact, and work on a fix or mitigation
- We may ask for additional details or help reproducing the issue

If the report is accepted, we will coordinate remediation and disclosure with the reporter where possible.

If the report is declined, we will explain why, for example if the issue is out of scope, not reproducible, or does not create a security impact.

## Disclosure Guidance

Please allow time for investigation and remediation before making details public. Coordinated disclosure helps protect users while a fix is being prepared and released.
